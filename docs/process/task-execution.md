# タスク実行フロー（エージェント非依存）

この文書は、`AGENTS.md` の長大タスク運用ルールを実務で使うための補助手順です。特定ツールや特定ベンダーの機能に依存しません。

## 1. 開始前

1. 対応ブランチを確認する。
2. `plans/<task-id>.md` を作成または更新する（`plans/TEMPLATE.md` を利用）。
3. `目的`、`非目標`、`対象範囲`、`マイルストーン`、`受け入れ条件`、`検証コマンド`、`既知 Blocker` を埋める。
4. 影響範囲の `AGENTS.md`（ルート + サブディレクトリ）を確認する。

## 2. 実行中

- マイルストーン単位で実装し、完了ごとに検証コマンドを実行する。
- 未ブロックで受け入れ条件に未達なら、次のマイルストーンへ継続する。
- 中間共有は計画ファイル更新、チェックリスト更新、ローカルコミットを基本とする。
- 通常 PR は受け入れ条件達成後に作成し、途中共有が必要な場合だけ Draft PR を使う。

## 3. 停止してよい条件

停止は原則として以下のいずれかに限る。

- 外部依存、権限不足、環境障害などで Blocked。
- 要件衝突があり、合理的な仮定だけでは進行不能。
- 破壊的変更の是非を現行文書だけでは判断できない。

## 4. 停止時の状態整合

停止時は `plans/<task-id>.md` で以下を必ず更新する。

- マイルストーン状態を `Done / Blocked / Cancelled` のいずれかにする（`pending` / `in_progress` を残さない）。
- `Blocked` の場合は停止理由と再開条件を明記する。
- 「次の最短アクション」を 1 行で残す。

## 5. 完了

- 受け入れ条件をすべて満たし、必要な検証コマンドが通過したら完了とする。
- 完了後に通常 PR を作成する。
